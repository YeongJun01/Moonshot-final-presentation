프로젝트 개요

본 프로젝트는 노션과 유사한 프로젝트 일정 관리 서비스를 구현하는 것을 목표로 하며, 요구사항 분석을 통해 프로젝트 및 할 일 관리 기능을 설계하고, 구글 로그인과 캘린더 연동을 포함한 핵심 기능을 구현했습니다.

개발 환경 및 사용 스택

  • Backend: Node.js, Express
  • Database: PostgreSQL
  • Deployment: Render, Netlify
  • API 문서화: Swagger
  • 협업 도구: GitHub, Notion, Discord

팀 구성 및 역할

  • 김지수 – 댓글
  • 나영준 – 프로젝트 설계, 인증, 파일 업로드, 할 일
  • 박건용 – 프로젝트, 멤버, 댓글, 태그, 하위 할 일, 문서화
  • 이광수 – 하위 할 일
  • 이상휘 – 할 일

협업 방식

  • Git Flow 기반 브랜치 전략
  • 주 2회 회의 진행
  • 이슈 단위 개발 및 코드 리뷰

Controller → Service → Repository
요청 / 비즈니스 로직 / DB 접근 계층 분리

- Controller
  · HTTP 요청/응답 처리
  · 입력값 검증 및 예외 처리

- Service
  · 비즈니스 로직 담당
  · 트랜잭션 관리

- Repository
  · DB 접근 전담
  · Prisma 기반 데이터 처리

배포 다이어그램

ERD

트러블 슈팅

  • 문제: OAuth 인증 후 토큰 전달 오류
  • 원인: 쿠키 설정 및 CORS 정책
  • 해결: HTTP Only 쿠키 적용

회고

  • 잘한 점: 역할 분담과 구조 설계
  • 아쉬운 점: 초기 일정 관리
  • 느낀 점: 사전 설계의 중요성
